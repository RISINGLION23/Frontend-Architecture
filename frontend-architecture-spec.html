Shared Principles
PatternOptimistic UI first — update local state immediately, rollback on API error. Never block the user waiting for a network round-trip.
PerfURL as source of truth — filters, pagination, search query all live in URL params. Deep-linkable, shareable, back-button safe.
SecurityNever trust the client — validate on both sides. Client validation for UX speed, server validation for integrity.
ErrorError boundaries at every boundary — wrap each major section so one widget crash can't destroy the whole page.
StateFinite state machines — every async operation has exactly: idle → loading → success | error. No boolean soup.
Tech Stack Assumptions
FrameworkReact 18 + TypeScript 5. Concurrent features (useTransition, useDeferredValue) used where applicable.
DataTanStack Query v5 for server state. Zustand for ephemeral client state. No Redux.
FormsReact Hook Form + Zod for validation. Uncontrolled inputs, schema-driven error messages.
AuthHTTP-only cookies for tokens. No localStorage. Refresh token rotation. PKCE for OAuth.
RouterTanStack Router (type-safe) or Next.js App Router. URL state via nuqs.

Step validation model
// Per-field validation layers

Layer 1: Inline Zod schema
  onChange → parse field value
  onBlur   → mark field as touched
  Display error only if touched

Layer 2: Step schema validation
  On NEXT click → parse full step
  Collect ALL errors at once
  Scroll to first error

Layer 3: Async cross-field checks
  After local validation passes
  e.g. email uniqueness, slug exists
  Debounced 400ms on change

Layer 4: Server validation on submit
  Map 422 field errors → RHF errors
  setError('fieldName', {message})
  Jump to offending step if needed

Error classification
FieldInline validation errors — displayed beneath field, only after field is touched. Never show errors on a pristine field. Clear on fix.
StepStep summary errors — shown in banner at top of step when NEXT is clicked and multiple fields are invalid. List all errors.
AsyncServer validation errors (422) — map field paths from API response to RHF setError(). Jump back to offending step if on review screen.
NetworkSubmit network failure — show retry UI, preserve all form data. Never lose user input on a transient error.
SessionSession expired mid-form — intercept 401, redirect to login with return URL, restore draft on return via persistKey.

Optional stepConditional steps that don't apply show a "Skip this section" option instead of empty fields.
BACK navigationValues must be preserved when going back. RHF preserves values by default, but custom components (rich text, file upload) need explicit state management. Test NEXT → BACK → NEXT round-trip.
Browser back buttonBrowser back while on step 3 should go to step 2 (not leave the form entirely). Sync step index to URL hash (#step-2). Intercept popstate event.
Tab close / refreshbeforeunload event — warn user if form has unsaved changes and no draft persistence. "You have unsaved changes. Leave?" Browser native dialog, cannot be customized.
Double submitRace condition: user clicks submit twice quickly. Disable submit button on first click. Use AbortController to cancel duplicate in-flight requests. Idempotent API endpoint with client-generated request ID.
Conditional stepsStep 3 only shows if checkbox on step 1 is checked. Re-evaluate step visibility on every field change. Clear data from hidden steps before submit. Don't include hidden step fields in final payload.
Large file uploadFile field on step 2, user is on step 4. File should upload in background (pre-upload pattern). If user goes back and changes the field, cancel the pending upload and restart.
Autofill conflictBrowser autofills address fields across steps. RHF uncontrolled inputs may not capture autofilled values. Force trigger('fieldName') on step activation for fields known to be autofilled.
Session expiryUser is on step 4, session expires (30 min idle). On submit, catch 401. Persist draft to localStorage (not session). Redirect to login with ?returnTo. On return, restore from localStorage and show "Welcome back" notification.

// components/MultiStepForm/
// ├── index.tsx                 ← public export
// ├── FormOrchestrator.tsx      ← state machine, routing between steps
// ├── FormProgress.tsx          ← progress bar + step indicators
// ├── StepRenderer.tsx          ← dynamically renders current step
// ├── ReviewStep.tsx            ← read-only summary
// ├── SubmitError.tsx           ← retry UI
// ├── useFormOrchestrator.ts    ← core state machine hook
// ├── useDraftPersistence.ts    ← sessionStorage read/write
// ├── useStepValidation.ts      ← async step-level validation
// ├── formReducer.ts            ← useReducer actions/reducer
// ├── types.ts                  ← StepConfig, FormState, etc.
// └── schemas/                  ← Zod schemas per step

// ─── Core types ────────────────────────────────────────────────
interface StepConfig<T> {
  id: string;
  title: string;
  fields: (keyof T)[];
  schema: ZodSchema;
  component: React.ComponentType<{ control: Control<T> }>;
  condition?: (values: Partial<T>) => boolean;
  validateAsync?: (data: Partial<T>) => Promise<Record<string, string>>;
}

type FormMachineState =
  | { status: 'idle' }
  | { status: 'active'; step: number; visited: Set<number> }
  | { status: 'validating'; step: number }
  | { status: 'submitting' }
  | { status: 'submit_error'; error: Error }
  | { status: 'complete' };

// ─── FormOrchestrator ──────────────────────────────────────────
export function MultiStepForm<T extends FieldValues>({
  steps, initialValues, onSubmit, onStepChange, persistKey, validateOnStep
}: MultiStepFormProps<T>) {
  const methods = useForm<T>({
    resolver: zodResolver(mergeSchemas(steps)),
    defaultValues: initialValues,
    mode: 'onBlur',
    criteriaMode: 'all',
  });

  const [state, dispatch] = useReducer(formReducer, { status: 'idle' });
  const { restoreDraft, saveDraft, clearDraft } = useDraftPersistence(persistKey);

  const activeSteps = useMemo(() =>
    steps.filter(s => !s.condition || s.condition(methods.watch())),
    [steps, methods.watch()]
  );

  const handleNext = async () => {
    if (state.status !== 'active') return;
    const currentStep = activeSteps[state.step];
    const valid = await methods.trigger(currentStep.fields);
    if (!valid) { scrollToFirstError(); return; }

    dispatch({ type: 'VALIDATING' });
    try {
      const errors = await currentStep.validateAsync?.(methods.getValues());
      if (errors && Object.keys(errors).length) {
        Object.entries(errors).forEach(([k, v]) =>
          methods.setError(k as Path<T>, { message: v }));
        dispatch({ type: 'VALIDATE_FAIL' });
      } else {
        dispatch({ type: 'NEXT', maxStep: activeSteps.length - 1 });
        onStepChange?.(state.step + 1);
      }
    } catch { dispatch({ type: 'VALIDATE_FAIL' }); }
  };

  const handleSubmit = methods.handleSubmit(async (data) => {
    dispatch({ type: 'SUBMITTING' });
    try {
      await onSubmit(stripHiddenSteps(data, activeSteps));
      clearDraft();
      dispatch({ type: 'SUCCESS' });
    } catch (err) {
      if (err instanceof ValidationError) mapServerErrors(err, methods, activeSteps);
      dispatch({ type: 'ERROR', error: err as Error });
    }
  });

  return (
    <FormProvider {...methods}>
      <FormProgress steps={activeSteps} current={state.step} visited={state.visited} />
      <ErrorBoundary fallback={<FormCrashFallback />}>
        {state.status === 'complete'
          ? <SuccessScreen />
          : state.status === 'submit_error'
          ? <SubmitError error={state.error} onRetry={handleSubmit} />
          : state.step === activeSteps.length
          ? <ReviewStep steps={activeSteps} onEdit={dispatch} onSubmit={handleSubmit} />
          : <StepRenderer step={activeSteps[state.step]} onNext={handleNext} onBack={() => dispatch({ type: 'BACK' })} isValidating={state.status === 'validating'} />
        }
      </ErrorBoundary>
    </FormProvider>
  );
}

  basePrice * qty

Invalid couponDistinguish: invalid code / expired / wrong plan / usage limit hit. Show specific message per case. Don't just say "Invalid code."

// ─── usePricingCalculator hook ─────────────────────────────────
function usePricingCalculator({
  plan, qty, addons, coupon, currency, locale, endpoint
}: PricingParams) {
  // Layer 1: instant local computation
  const localPrice = useMemo(() =>
    computeLocalPrice({ plan, qty, addons }),
    [plan, qty, addons]
  );

  // Defer rendering so slider drag stays smooth
  const deferredParams = useDeferredValue({ plan, qty, addons, coupon });
  const isPending = deferredParams !== { plan, qty, addons, coupon };

  // Layer 2: debounced server sync
  const { data: serverPrice, isLoading, isError } = useQuery({
    queryKey: ['price', deferredParams],
    queryFn: () => fetchPrice(endpoint, deferredParams),
    staleTime: 5 * 60 * 1000,  // 5 min — matches token TTL
    placeholderData: keepPreviousData,
    retry: 2,
  });

  const displayPrice = serverPrice?.total ?? localPrice;
  const isEstimated = !serverPrice || isPending || isLoading;

  const formatter = useMemo(() =>
    new Intl.NumberFormat(locale, {
      style: 'currency', currency,
      minimumFractionDigits: 2,
    }),
    [locale, currency]
  );

  return {
    displayPrice: formatter.format(displayPrice / 100),
    isEstimated,
    isError,
    breakdown: serverPrice?.breakdown ?? [],
    priceToken: serverPrice?.token,
    couponResult: serverPrice?.coupon,
  };
}

// ─── PricingCalculator component ───────────────────────────────
export function PricingCalculator({ plans, currency, locale, onPriceConfirmed, pricingEndpoint }) {
  const [params, setParams] = useState<PricingParams>({
    plan: plans[0], qty: 1, addons: [], coupon: null
  });

  const pricing = usePricingCalculator({ ...params, currency, locale, endpoint: pricingEndpoint });

  return (
    <div>
      <PlanSelector plans={plans} selected={params.plan} onChange={p => setParams(prev => ({...prev, plan: p}))} />
      <QuantityInput value={params.qty} onChange={q => setParams(prev => ({...prev, qty: q}))} />
      <AddonSelector addons={params.addons} onChange={a => setParams(prev => ({...prev, addons: a}))} />
      <CouponField onApply={c => setParams(prev => ({...prev, coupon: c}))} result={pricing.couponResult} />
      <PriceDisplay
        price={pricing.displayPrice}
        isEstimated={pricing.isEstimated}
        isError={pricing.isError}
        breakdown={pricing.breakdown}
      />
      <CheckoutButton
        disabled={pricing.isEstimated || pricing.isError}
        onClick={() => pricing.priceToken && onPriceConfirmed(pricing.priceToken)}
      />
    </div>
  );
}

// ─── useSearch hook ────────────────────────────────────────────
function useSearch() {
  const [isPending, startTransition] = useTransition();

  // All state from URL — zero useState for search params
  const [q] = useQueryState('q');
  const [categories] = useQueryState('category', parseAsArrayOf(parseAsString));
  const [sort] = useQueryState('sort', parseAsString.withDefault('relevance'));
  const [page] = useQueryState('page', parseAsInteger.withDefault(1));

  const searchParams = useMemo(() => ({
    q: q ?? '', categories: categories ?? [], sort, page
  }), [q, categories, sort, page]);

  const { data, isLoading, isFetching, isError, isPreviousData } = useQuery({
    queryKey: ['search', searchParams],
    queryFn: ({ signal }) => searchAPI(searchParams, signal),
    placeholderData: keepPreviousData,
    staleTime: 30_000,
  });

  return {
    results: data?.items ?? [],
    facets: data?.facets ?? [],
    total: data?.total ?? 0,
    isLoading: isLoading && !isPreviousData,
    isFiltering: isFetching || isPending,
    isError,
    isEmpty: !isLoading && data?.total === 0,
    hasActiveFilters: (categories?.length ?? 0) > 0,
    startTransition,
  };
}

      />

        }
        <Pagination total={search.total} />
      </main>
    </div>
  );
}

const updateMutation = useMutation({
  mutationFn: updateResource,

  onMutate: async (newData) => {
    // 1. Cancel any in-flight fetches for this query
    await queryClient.cancelQueries({ queryKey: ['resources'] });
    // 2. Snapshot previous value for rollback
    const prev = queryClient.getQueryData(['resources']);
    // 3. Optimistically update cache immediately
    queryClient.setQueryData(['resources'], applyUpdate(prev, newData));
    return { prev };  // returned as context
  },

  onError: (err, vars, context) => {
    // 4. Rollback on error
    queryClient.setQueryData(['resources'], context.prev);
    toast.error(`Failed: ${err.message}`);
  },

  onSettled: () => {
    // 5. Always refetch after mutation (success or error)
    queryClient.invalidateQueries({ queryKey: ['resources'] });
  },
});

// ─── WebSocket real-time hook ──────────────────────────────────
function useDashboardRealtime() {
  const queryClient = useQueryClient();
  const [status, setStatus] = useState<'connecting'|'live'|'stale'>('connecting');

  useEffect(() => {
    const ws = new WebSocket(`${WS_URL}/dashboard`);
    const batchQueue: WSDelta[] = [];
    let rafId: number;

    const flushBatch = () => {
      if (batchQueue.length === 0) return;
      const deltas = [...batchQueue];
      batchQueue.length = 0;
      // Apply all deltas to query cache in one batch
      queryClient.setQueriesData({ queryKey: ['dashboard'] },
        (old: any) => applyDeltas(old, deltas));
    };

    ws.onmessage = (event) => {
      batchQueue.push(JSON.parse(event.data));
      cancelAnimationFrame(rafId);
      rafId = requestAnimationFrame(flushBatch);  // max 60fps
    };

    ws.onopen = () => setStatus('live');
    ws.onclose = () => setStatus('stale');

    return () => { ws.close(); cancelAnimationFrame(rafId); };
  }, [queryClient]);

  return { status };
}

Token strategy
Access token:
  HTTP-only cookie, Secure, SameSite=Lax
  Short-lived: 15 minutes
  Never accessible to JavaScript
  Set by server on /api/auth/login

Refresh token:
  HTTP-only cookie, Secure, SameSite=Strict
  Long-lived: 30 days
  Stored separately from access token
  Rotated on every use (rotation strategy)

Silent refresh:
  React Query refetchInterval: 14 min
  Calls GET /api/auth/refresh
  Server rotates refresh token
  Returns new access token cookie
  Invisible to user

PKCE OAuth flow
// Google / GitHub SSO with PKCE

1. User clicks "Continue with Google"

2. Client generates:
   codeVerifier = crypto.randomUUID() + ...
   codeChallenge = base64url(sha256(verifier))
   state = crypto.randomUUID()  // CSRF

3. Store { state, codeVerifier } in
   sessionStorage  // survives redirect

4. Redirect to Google with:
   ?response_type=code
   &code_challenge={codeChallenge}
   &code_challenge_method=S256
   &state={state}

5. Google redirects back with ?code&state
6. Verify state matches sessionStorage
7. POST /api/auth/oauth { code, verifier }
8. Server exchanges code for tokens
9. Server sets HTTP-only cookies
10. Clear sessionStorage, redirect app

Expired reset tokenUser clicks old reset link: detect on page load (verify token before showing form). Show "This link has expired. Request a new one." Don't show the password form at all — confusing to fill then fail.

// ─── AuthProvider ──────────────────────────────────────────────
function AuthProvider({ children }) {
  const { data: session, status } = useQuery({
    queryKey: ['session'],
    queryFn: fetchSession,        // GET /api/auth/me
    retry: false,
    staleTime: 14 * 60 * 1000,    // 14 min
    refetchInterval: 14 * 60 * 1000, // silent refresh
    refetchIntervalInBackground: false,
  });

  const logout = async () => {
    await fetch('/api/auth/logout', { method: 'POST' });
    queryClient.clear();      // clear ALL cached user data
    router.navigate('/login');
  };

  return (
    <AuthContext.Provider value={{ session, status, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

// ─── ProtectedRoute ────────────────────────────────────────────
function ProtectedRoute({ children, requiredRole }: ProtectedRouteProps) {
  const { session, status } = useAuth();
  const location = useLocation();

  if (status === 'pending') return <AuthLoadingSpinner />;

  if (!session) {
    // Save intended destination for post-login redirect
    return <Navigate to={`/login?returnTo=${encodeURIComponent(location.pathname)}`} replace />;
  }

  if (requiredRole && !session.roles.includes(requiredRole)) {
    return <Navigate to="/unauthorized" replace />;
  }

  return <>{children}</>;
}

// ─── useAuthForm hook ──────────────────────────────────────────
function useAuthForm<T extends FieldValues>(
  schema: ZodSchema<T>,
  mutationFn: (data: T) => Promise<AuthResponse>,
  onSuccess: (response: AuthResponse) => void
) {
  const form = useForm<T>({
    resolver: zodResolver(schema),
    mode: 'onBlur',
  });

  const mutation = useMutation({
    mutationFn,
    onSuccess: onSuccess,
    onError: (err: AuthError) => {
      if (err.code === 'FIELD_ERRORS') {
        Object.entries(err.fields).forEach(([k, v]) =>
          form.setError(k as Path<T>, { message: v })
        );
      }
      // Rate limit, network errors handled by form state
    },
  });

  return {
    form,
    onSubmit: form.handleSubmit(mutation.mutate),
    isPending: mutation.isPending,
    isRateLimited: mutation.error?.code === 'RATE_LIMITED',
    retryAfter: mutation.error?.retryAfter,
  };
}
